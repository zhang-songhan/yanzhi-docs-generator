# yanzhi-docs-generator

一个 Claude Code 插件，用于从源代码自动生成和维护详细的项目文档。

## 功能

- **全量生成**：扫描代码库，生成完整的 `docs/` 目录，包含 PRD、架构文档、数据模型、模块详细设计等。
- **增量更新**：当源代码变更时，仅更新受影响的文档，保留手动编辑的内容。
- **AI 可重建保真度**：每份项目文档的详细程度足以让另一个 AI 代理仅凭文档即可从零重建整个项目。
- **全中文输出**：所有文件名、目录名和文档内容均使用简体中文。
- **Mermaid 图表**：逻辑流程使用图表表达（flowchart、sequence、ER、state、class），绝不包含原始源代码。
- **前端 UI 截图占位符**：前端/UI 模块文档自动插入截图占位符（`【图X：...】`），附带图片引用链接指向同级 `截图/` 目录。
- **Web App 自动截图**：检测到 Web App 项目时，自动调用 `auto-capture-for-webapp:take-screenshots` Skill 启动项目并截取页面截图，保存到对应的 `截图/` 目录。
- **修改历史追踪**：每份文档记录修改时间、版本号、变更内容和操作人。

## 安装

### 1. 添加插件市场

```
/plugin marketplace add zhang-songhan/yanzhi-docs-generator
```

### 2. 安装插件

```
/plugin install yanzhi-docs-generator
```

### 3. 安装截图自动捕获依赖（可选）

如需自动截取 Web App 截图，需安装 auto-capture-for-webapp 插件：

```
/plugin install auto-capture-for-webapp@auto-capture-for-webapp
```

## 使用

运行 `/writing-docs` 即可生成或更新项目文档：

```
/writing-docs
```

### 工作模式

| 场景 | 行为 |
|------|------|
| `docs/` 目录不存在 | 全量生成 — 扫描整个代码库，从零构建 `docs/` |
| `docs/` 已有文档 | 增量更新 — 使用 `git diff`（最近 3 次提交）检测变更，仅更新受影响的文件 |
| 无有意义的源代码变更 | 报告无需更新并退出 |

### 生成的目录结构

```
docs/
├── README.md                      # 导航索引
├── 01_产品需求文档/               # PRD：系统概述、能力清单、用户故事、非功能需求
├── 02_核心业务流程/               # 端到端业务流程（Mermaid 图）
├── 03_系统架构与开发/             # 系统架构、数据模型、模块集成
├── 04_运行时与部署/               # 运行时配置、部署与运维
├── 05_实现指南/                   # 端到端重建蓝图、模块目录与覆盖率矩阵
└── 06_模块详细设计/               # 每个模块的详细设计文档及子模块说明
    ├── 01_认证模块/
    │   ├── 01_模块概述.md
    │   ├── 02_登录流程.md
    │   └── 截图/                  # UI 模块的截图目录
    │       ├── 1-login-page.png
    │       └── 2-login-error.png
    └── 03_用户界面/
        ├── 01_模块概述.md
        ├── 02_仪表盘.md
        └── 截图/
            └── 1-dashboard-overview.png
```

### 截图占位符格式

前端 UI 文档中自动插入如下占位符：

```
【图1：登录页面 - 默认状态，展示系统Logo、用户名输入框、密码输入框、登录按钮的布局】

![图1](截图/1-login-page.png)
```

### 文档生成完成后

若包含前端 UI 模块且满足 `auto-capture-for-webapp:take-screenshots` Skill 的截图条件，会自动为每个截图占位符捕获真实截图。若无法自动截图，占位符会保留在文档中供手动补充。

### 支持的编程语言

项目文档从以下源代码文件中提取：`.py`、`.html`、`.js`、`.ts`、`.tsx`、`.jsx`、`.java`、`.sh`、`.go`、`.rs`、`.php`、`.rb`、`.cs`、`.kt`、`.swift`、`.sql`、`.yaml`、`.yml`。

## 核心规则

- **不包含源代码** — 所有逻辑通过 Mermaid 图表和自然语言描述表达。
- **中文编号命名** — `docs/` 下所有文件和目录使用 `{序号}_{中文名称}` 格式。
- **保留手动编辑** — 增量更新仅修改与源代码变更对应的部分。
- **修改历史** — 每份文档以修改历史表开头，记录每次变更。
- **前端截图占位符** — 前端/UI 模块文档插入截图占位符，Web App 项目通过 `auto-capture-for-webapp:take-screenshots` Skill 自动截取并保存截图。
- **截图目录** — 截图保存在对应文档同级 `截图/` 目录中。
